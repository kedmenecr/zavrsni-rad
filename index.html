Otvaranje novog direktorija

Napravite novu mapu te u toj mapi napravite
git init
za otvaranje novog git repozitorija.

Checkout repozitorija

Stvorite radnu kopiju lokalnog repozitorija upisivanjem komanda
git clone /path/to/repository
kod korištenja remote servera vaša komanda će se zvati
git clone username@host:/path/to/repository

Radni tijek

Vaš lokalni repozitorij se sastoj od tri grane održavane od strane gita. Prva je vaš Working Directory koji sadržava aktualne datoteke. drugi je Index koji služio kao stagging prostor i na kraju HEAD koji pokazuje zadnji commit na kojem ste radili.

add & commit

Možete predložiti promjene (dodati ih na Index) koristeći
git add <filename>
or
git add *
To su prvi koraci ka osnovnome git workflowu. Da bi uopče commitali te promjene moramo koristiti
git commit -m "Commit message"
Sad su promjene commitane na HEAD, ali još nisu na remotnome repozitoriju.

Pushanje promjena

Trenutno se nalazite u HEAD od vaše kopije radnog direktorija. Za slanje tih promjena k remote repositoriju ( server )
git push origin master
Promjenite master u bilokoji branch na koji želite pushat

Ako niste klonirali postoječi repozitorij i želite povezate svoj repozitorij s remote serverom, morate ga pomoću ove komande
git remote add origin <server>
Sada možete poslati svoje promjene na remote server

branching

Branchevi su korišteni za razvoj dodatnih svojstva izoliranih jednog od drugog. master branch je "defaultni" branch kad se kreira repozitorij. Ostali branchevi se koristi za razvoj i onda se mergaju s master branchu na kraju.

Napravite novi branch zvan "feature_x" i promjeniti se na taj branch koristeči
git checkout -b feature_x
za vračanje na master opet,koristite
git checkout master
i za brisanje tog brancha koristimo
git branch -d feature_x
branch nije dostupan drugima osim ako se ne pusha na remote repozitorij
git push origin <branch>

update & merge

Za update vašeg lokalnog repozitorija do najnovijeg commita koristite
git pull
za update vašeg radnog direktorija fetch i merge nove promjene.
Za merganje drugog brancha u vaš aktivni branch (npr. master), koristite
git merge <branch>
U oba slučaja git želi auto-mergate promjene. Nažalost to nije uvjek moguče i rezultira u conflicts Vi ste odgovorni za manualno merganje conflicts tako što čete editati datoteke prikazane od strane gita. Poslije toga morate obilježiti sa merged with
git add <filename>
Prije nego mergate promjene, možete napraviti pregled prije sa
git diff <source_branch> <target_branch>

Korisni trikovi

ugrađen git GUI
gitk
obojen git output
git config color.ui true
Prikaz cijelog loga u jednoj liniji po commit
git config format.pretty oneline
use interactive adding
git add -i
